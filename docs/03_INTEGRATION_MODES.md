# 03. Integration modes

## Mode 1: observe-only

SDK только собирает события и не закрывает контент.

Подходит для первого аудита и клиента, который боится ухудшить UX.

Плюсы:
- почти нет риска конфликтов;
- можно собрать базовые цифры;
- легко поставить.

Минусы:
- не защищает контент.

## Mode 2: soft-gate

SDK скрывает выбранный блок контента до проверки рекламной видимости.

Плюсы:
- быстро показывает продуктовую ценность;
- подходит для демо;
- не требует backend клиента.

Минусы:
- frontend-only защита может быть обойдена продвинутым пользователем.

## Mode 3: server-gate

Рекомендованный режим. Backend клиента отдаёт полный контент только после проверки visitorToken через Ваш SaaS.

Плюсы:
- защищённый контент не должен лежать в DOM заранее;
- лучше подходит для файловых страниц, каталогов, premium/free content, download pages;
- проще объяснить бизнес-ценность.

Минусы:
- нужен backend-разработчик клиента;
- интеграция дольше;
- нужно аккуратно продумать fallback.
