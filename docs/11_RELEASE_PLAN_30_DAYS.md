# 11. 30-day beta release plan

## Week 1 — local + staging

- перевірити локальний запуск
- змінити ADMIN_PASSWORD
- підключити домен
- поставити VPS
- поставити reverse proxy
- підключити HTTPS
- протестувати dashboard
- створити 3–5 demo projects

## Week 2 — integration package

- підготувати короткий onboarding PDF/Google Doc
- підготувати лист для клієнта
- підготувати WordPress/GTM інструкцію
- протестувати observe-only
- протестувати soft-gate
- перевірити allowed domains

## Week 3 — QA + hardening

- browser matrix
- extension matrix
- manual DOM tampering
- weak connection simulation
- server-gate prototype
- export CSV
- backup data file

## Week 4 — beta outreach

- 50–100 потенційних сайтів
- перший лист: observe-only, без блокування
- ціль: 10–20 відповідей
- ціль: 3–5 реальних інтеграцій
- після 7 днів показати dashboard summary

## Коли можна показувати демку

Демку можна показувати вже після того, як:

- SaaS стоїть на HTTPS-домені
- dashboard працює
- є demo project
- snippet вставляється у demo customer page
- overlay і content_unlocked пишуться в analytics
- Ви можете за 3 хвилини пояснити observe-only і soft-gate
