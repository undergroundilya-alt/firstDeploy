# Privacy notice template for beta clients

This is a draft template, not legal advice.
Before commercial launch, adapt it with a lawyer for your jurisdiction and customer contracts.

## What the service does

The service verifies whether an advertising container is delivered and visibly rendered before protected content is made available.

## Data processed

The beta may process:

- project public key;
- page URL or origin;
- browser user-agent string;
- pseudonymous visitor token;
- event type and reason;
- timestamp;
- hashed IP-derived fingerprint for abuse/rate-limit analytics;
- technical proof data such as canvas hash prefix and visibility ratio.

## What is not required

The beta does not require:

- name of the website visitor;
- email address of the website visitor;
- payment data of the website visitor;
- precise geolocation;
- content of private messages.

## Retention

Define retention per customer agreement. For beta testing, keep event data only as long as needed to validate integration, debug false positives and improve product reliability.

## Visitor communication

When the overlay appears, the website should clearly explain that access depends on allowing the advertising container on that site, and should avoid misleading or aggressive wording.
