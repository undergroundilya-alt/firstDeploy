# 08. Licenses, certificates and customer access

## Нужна ли “лицензия” клиенту

Для beta это не государственная лицензия и не сертификат. В продуктовой модели “лицензия” — это запись проекта клиента в SaaS:

- company;
- project;
- public key;
- secret key;
- allowed domains;
- integration mode;
- срок beta-доступа;
- лимиты событий.

То есть клиент получает не файл-лицензию, а доступ к сервису и ключи интеграции.

## Нужен ли сертификат клиента

Обычно нет.

TLS-сертификат нужен Вашему SaaS-домену, например:

```text
https://avp.example.com/sdk/v1/PROJECT_PUBLIC_KEY.js
```

Клиентский сайт уже должен работать по HTTPS. Браузер просто загружает Ваш SDK по HTTPS.

## Когда может понадобиться custom domain

Иногда крупный клиент захочет подключать SDK с поддомена своего домена:

```text
https://ad-verify.client.com/sdk/v1/PROJECT_PUBLIC_KEY.js
```

Тогда возможны варианты:

1. Клиент создаёт CNAME на Вашу инфраструктуру.
2. Вы выпускаете TLS через Let’s Encrypt/Cloudflare.
3. Или клиент сам проксирует SDK через свой сервер.

Для beta это не обязательно. Начните с Вашего SaaS-домена.

## Какой доступ нужен к сайту

### Минимальный доступ

- вставить JS snippet перед `</body>`;
- добавить рекламный контейнер;
- указать selector защищённого контента.

Это можно сделать через CMS, шаблон сайта, Google Tag Manager или силами разработчика клиента.

### Расширенный доступ

- изменить шаблон страницы;
- вынести защищённый контент из HTML до проверки;
- подключить backend verify API;
- сделать fallback-страницу.

### Что не стоит просить без необходимости

- полный SSH;
- полный FTP;
- доступ к базе данных;
- root-доступ;
- доступ к рекламному аккаунту клиента;
- доступ к платежам клиента.

Лучше дать клиенту понятную техническую задачу, а не просить полный контроль.

## Что подписывать с beta-клиентом

Минимально:

1. Beta terms / письмо-согласие на тест.
2. Privacy / data processing notice.
3. Список собираемых событий.
4. Срок теста: 30 дней.
5. Право клиента отключить snippet в любой момент.
6. Ограничение ответственности: beta, no SLA.
7. Запрет раскрывать secret key публично.

Для крупных клиентов:

- NDA;
- DPA;
- техническое приложение;
- security questionnaire;
- отдельная интеграционная спецификация.
