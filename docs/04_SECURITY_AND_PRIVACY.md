# 04. Security and privacy baseline

## Что хранит beta

- project id;
- public key;
- secret key;
- домены проекта;
- visitor token;
- page URL;
- origin;
- user-agent;
- хешированный IP/user-agent fingerprint;
- тип события;
- причина события;
- timestamp.

## Что лучше не хранить

- полный IP в открытом виде;
- email или персональные данные посетителя сайта;
- содержимое страниц клиента;
- cookies клиента;
- рекламные revenue-данные без отдельного согласования;
- чувствительные URL, если они содержат персональные данные.

## Что нужно для реального beta-теста

1. Privacy notice / Data Processing Addendum для клиентов.
2. Retention period: например 30–90 дней для сырых событий.
3. Агрегация старых данных.
4. Удаление проекта по запросу клиента.
5. Ограничение доступа к dashboard.
6. Secret key не вставлять во frontend.
7. HTTPS обязательно.
8. Разделение production/staging.
9. Rate limit на API.
10. Audit log для действий администратора.

## Важное ограничение

Система не должна мотивировать пользователя к клику по рекламе. Продукт проверяет видимость рекламного контейнера и доступ к контенту, а не стимулирует клики или просмотры ради вознаграждения.
