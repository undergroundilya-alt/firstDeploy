# 05. Roadmap на 30 дней

## Неделя 1 — поднять beta SaaS локально и на staging

- Запустить проект локально.
- Поменять ADMIN_PASSWORD.
- Настроить публичный домен или staging subdomain.
- Поставить reverse proxy.
- Подключить нормальный TLS.
- Проверить dashboard.
- Создать 3–5 тестовых проектов.
- Протестировать demo customer page.

## Неделя 2 — довести интеграции

- Сделать 2 режима onboarding: observe-only и soft-gate.
- Подготовить инструкцию для WordPress/CMS/GTM.
- Подготовить server-gate пример для Node/PHP/Python.
- Добавить нормальные тексты overlay.
- Добавить экспорт CSV событий.
- Добавить фильтр dashboard по проекту/датам.

## Неделя 3 — тестирование и стабильность

- Проверить Chrome, Firefox, Edge, мобильные браузеры.
- Проверить AdBlock/uBlock обычные сценарии.
- Проверить плохой интернет / offline / DNS issues.
- Проверить, не ломает ли SDK сайт.
- Проверить CORS и allowed domains.
- Проверить 20–50 тысяч событий на тестовой нагрузке.

## Неделя 4 — beta launch

- Подготовить landing/pitch.
- Подготовить one-page offer.
- Подготовить beta agreement.
- Подготовить onboarding email.
- Найти 50–100 сайтов.
- Начать с observe-only для осторожных клиентов.
- Для 5–10 самых заинтересованных предложить server-gate.

## Что важно иметь перед показом клиенту

- стабильный dashboard;
- понятный install snippet;
- понятный список событий;
- демо-страница;
- честные ограничения beta;
- privacy/security пояснение;
- понятная фраза: “мы не заставляем кликать рекламу, мы проверяем рекламную видимость перед выдачей защищённого контента”.
