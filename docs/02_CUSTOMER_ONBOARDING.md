# 02. Customer onboarding для 50–100 компаний

## Что нужно получить от клиента

Минимальный вариант:

1. Домен сайта.
2. URL страниц, где есть рекламная монетизация.
3. Где находится рекламный блок или куда его можно поставить.
4. CSS selector защищённого контента.
5. Доступ к CMS / шаблону / Google Tag Manager / разработчику клиента для вставки snippet.
6. Понимание рекламной сети клиента: Google AdSense, Ad Manager, другая сеть, direct ads, affiliate block.

Строгий вариант:

1. Всё из минимального варианта.
2. Участие backend-разработчика клиента.
3. Endpoint или middleware на стороне клиента, который перед выдачей контента вызывает server-to-server verify.
4. Решение, что делать при `allowed: false`: показать preview, paywall-like message, retry screen, fallback.

## Нужен ли полный доступ к сайту клиента

Обычно нет.

Варианты доступа:

- клиент сам вставляет snippet;
- клиент даёт доступ в CMS;
- клиент даёт доступ в Google Tag Manager;
- клиент передаёт задачу своему разработчику;
- Вы даёте инструкцию и проверяете результат;
- для сложной интеграции клиент подключает backend-разработчика.

Полный FTP/SSH/admin доступ просить нежелательно. Это повышает риск и снижает доверие.

## Что дать клиенту

1. Install snippet.
2. Public key проекта.
3. Secret key только для backend-разработчика.
4. Список allowed domains.
5. Инструкцию, куда вставить контейнер рекламы.
6. Инструкцию по тестированию.
7. Контакт для технических вопросов.
8. Объяснение, что beta месяц бесплатная и нужна обратная связь.
